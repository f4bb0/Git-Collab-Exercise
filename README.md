## 📝 协作规范

同步：开始前先 git pull origin main。  
分支：创建新分支 feature/your-name (例如 feature/fabbo)。实际开发时分支名应该写清楚功能。  
提交：例如，git commit -m "feat: add user message #IssueID"。  
合并：本地尝试 git merge main 解决冲突后再推送到远程提 PR。因为我开启了 branch protection，不能直接 merge 到主分支，所以需要提 PR 并由其他成员审批。
